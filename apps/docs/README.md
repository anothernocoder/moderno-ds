# @moderno/docs

Bilingual (en/es) Astro docs for Moderno: MDX + Content Collections, Svelte
islands for live previews, an autogenerated `<PropsTable>`, a **Theme Builder**,
and the static registry served under `/r/`.

## Scripts

| Script              | What it does                                                     |
| ------------------- | ---------------------------------------------------------------- |
| `pnpm dev`          | Prebuild (props + registry) then `astro dev`.                    |
| `pnpm build`        | Prebuild → `astro build` → per-locale Pagefind index.            |
| `pnpm typecheck`    | `astro check`.                                                   |
| `pnpm check:parity` | Fail if any docs slug lacks an en/es translation (the CI guard). |

## Build pipeline

1. **prebuild** — `@moderno/props-doc` resolves the React types into
   `src/generated/props/*.json`; `scripts/copy-registry.ts` copies the repo
   `registry/` into `public/r/`.
2. **astro build** — static HTML, the `.md` twin per page, and `llms.txt` per
   locale.
3. **index** — Pagefind indexes `dist/en` and `dist/es` into separate bundles
   (`dist/<locale>/pagefind`) so search never mixes languages.

## i18n

Every page exists under `src/content/docs/<locale>/<slug>.mdx`. `check:parity`
(wired into CI as `pnpm docs:parity`) fails the build if a slug is missing in
either locale. Prop names and code identifiers stay in English in both locales;
only the surrounding labels are translated (`src/i18n/ui.ts`).

## Deploy (Vercel)

The `@astrojs/vercel` adapter emits `.vercel/output` (Build Output API v3).

- Set the Vercel project **Root Directory** to `apps/docs`.
- Domain: `moderno.style` (falls back to the generated Vercel URL until the
  domain is registered).
- `SITE_URL` env var overrides the canonical/`llms.txt` base URL for preview
  deploys; the CLI can point at a per-PR preview registry at `<url>/r/`.
