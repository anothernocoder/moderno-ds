Problem — the Token contract exists only as CSS declarations plus six parallel literal arrays; the Theme Builder’s GROUPS copy is unguarded, so a new slot makes theme compile require an input the editor never renders.

Solution — @moderno/tokens exports the contract as data (slot name, type, group, contrast pair); every other list becomes an import-and-derive.

  • locality: add a slot in one module
  • drift becomes impossible, not detected
  • editor can’t omit compiler-required slots
  • interface shrinks; one list replaces six
ADR-0001 already concedes the friction: “schema changes require updating builder controls, compile script, and the CONTRACT.md slot contract together.” This deepening retires that consequence rather than contradicting the ADR. Ship the data as source .ts/JSON via exports — no bundler step, consistent with the CSS-package toolchain decision.

5 · Bindings stop re-deriving what core and Ark own

Worth exploring in-process · small
packages/{vue,solid,svelte}/src/button.* · select.* (×7 files) · packages/react/src/select.tsx:31-45 · packages/core/src/recipes.ts

Before — the incantation, seven times

// repeated in vue, solid, svelte × button, select…
type ButtonVariant = NonNullable<
  VariantProps<typeof buttonRecipe.variants>["variant"]>;

// react/select.tsx: 13 Ark parts enumerated by hand
export const Select = { Root, Label: ArkSelect.Label,
  Control: …, Trigger: …, /* …9 more, drops any
  part Ark adds in a future version */ };

After — named types at the seam

// @moderno/core exports next to the recipes
export type { ButtonVariant, ButtonSize, SelectSize };

// every binding imports the name
import type { ButtonVariant } from "@moderno/core";

// react joins the spread the other three use
export const Select = { ...ArkSelect, Root };

Problem — each binding re-derives Component variant types with the same type-level incantation, and React’s Select hand-mirrors Ark’s part list — a shallow module that silently drops parts Ark adds later.

Solution — @moderno/core exports the derived variant types beside the recipes; React adopts the spread the other three bindings already use.

  • leverage: one derivation, N bindings
  • recipe rename ripples nowhere
  • drift-prone hand mirror deleted

Problem — which Primitives get a PropsTable is decided by an untested hardcoded list in a build script; a renamed export only surfaces at docs build time, and a new Primitive is silently undocumented.

Solution — colocate the manifest with the Primitives (or at minimum test that every entry resolves), keeping extractProps as the deep module it already is.

  • locality: new Primitive, one edit
  • broken wiring fails in tests, not builds

Top recommendation

Export the Token contract as data

Two independent explorations converged on it; ADR-0001 already names the friction as a consequence to live with. It is the cheapest of the strong candidates, it closes the one silent user-facing failure (a contract slot the compiler requires but the Theme Builder can’t edit), and it establishes the seam that candidates 3 and 6 then follow — the same shape the codebase’s best module, the shared theme-compile color engine, already models.
