# Platform, distribution, docs hosting, and theme authoring

Moderno is a solo-maintainer, shadcn-style design system. These decisions lock in the monorepo toolchain, how packages and registry items reach consumers, where docs and the registry JSON are hosted, docs i18n, and the Theme Builder shipped in v1.

## Monorepo toolchain

| Decision           | Choice                                                        | Rationale                                                                                    |
| ------------------ | ------------------------------------------------------------- | -------------------------------------------------------------------------------------------- |
| Package manager    | **pnpm 9+** workspaces                                        | Already chosen; efficient for monorepos                                                      |
| Node               | **22 LTS** (`engines.node >= 22`)                             | Current LTS; matches modern tooling defaults                                                 |
| Language           | **TypeScript 5.x**, strict                                    | Shared types across packages and docs scripts                                                |
| Module format      | **ESM-only** (`"type": "module"`, `exports` in every package) | Tree-shaking, no dual-package hazard; all targets support ESM                                |
| Library bundler    | **tsup** per publishable package                              | Fast, minimal config, good `.d.ts` emit                                                      |
| Tests              | **Vitest**                                                    | Same config story as Vite/Astro; fast unit tests for `@moderno/core`, `@moderno/charts-core` |
| Lint / format      | **ESLint** (typescript-eslint) + **Prettier**                 | Ecosystem familiarity; matches shadcn-adjacent projects                                      |
| Versioning         | **Changesets**                                                | Semver + changelog for `@moderno/*`; one PR can bump multiple packages                       |
| Task orchestration | **pnpm scripts only** (no Turborepo in v1)                    | Solo maintainer; add Turborepo only if build times hurt                                      |

CSS packages (`@moderno/tokens`, `@moderno/core`, `@moderno/css`) ship source `.css` via `exports` — no JS bundler step. JS/TS packages use tsup.

## npm publication

- **Scope:** public packages under `@moderno/*` on [npmjs.com](https://www.npmjs.com).
- **Visibility:** all v1 packages are **public**. No private registry in v1.
- **Versioning:** semver via Changesets. Breaking changes require a major bump on affected packages.
- **Changelog:** generated by Changesets at release time; also surfaced in GitHub Releases.
- **Publish trigger:** GitHub Actions on push to `main` when Changesets PR is merged (`changesets/action` → `pnpm publish -r`).
- **npm org:** `@moderno` org on npm; CI uses `NPM_TOKEN` secret.

**Published npm packages (v1):**

`@moderno/tokens`, `@moderno/core`, `@moderno/css`, `@moderno/charts-core`, `@moderno/react`, `@moderno/vue`, `@moderno/svelte`, `@moderno/solid`, `@moderno/cli`

**Not on npm (registry copy items):** blocks, themes, ejected primitives.

## Registry JSON (shadcn-style copy items)

Registry items (blocks, themes, eject) live in the repo under `registry/` and are **not** npm packages.

| Decision        | Choice                                                                                                                       |
| --------------- | ---------------------------------------------------------------------------------------------------------------------------- |
| Source of truth | `registry/registry.json` + item files in `registry/blocks/`, `registry/themes/`                                              |
| Public URL      | Served as static assets from the docs site: **`https://moderno.style/r/registry.json`**                                      |
| Item URLs       | `https://moderno.style/r/{type}/{name}.json` (e.g. `/r/themes/theme-moderno.json`)                                           |
| CLI default     | `@moderno/cli` defaults to the URL above; overridable via `components.json` → `registry` field or `MODERNO_REGISTRY_URL`     |
| Versioning      | Each registry item carries a semver; CLI manifest (`.moderno/manifest.json`) tracks installed versions for `update` / `diff` |
| Pre-release     | PR previews deploy docs + registry to Vercel preview URLs; CLI can point at preview registry for dogfooding                  |

Until `moderno.style` is registered, the CLI and docs use the **production Vercel deployment URL** as fallback (documented in README when the monorepo exists).

## Documentation site

| Decision          | Choice                                                                                                                                  | Rationale                                                              |
| ----------------- | --------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------- |
| Framework         | **Astro + MDX + Content Collections**                                                                                                   | Already chosen; no Starlight                                           |
| Preview runtime   | **Svelte islands only**                                                                                                                 | Already chosen; one client runtime                                     |
| Hosting           | **Vercel**                                                                                                                              | Zero-config Astro, preview deploys per PR, static `/r/` for registry   |
| Production domain | **`moderno.style`**                                                                                                                     | Short, product-aligned; DNS → Vercel                                   |
| Search            | **Pagefind** — **one index per locale** (`pagefind-en`, `pagefind-es`)                                                                  | Static, no SaaS; search stays language-scoped                          |
| i18n              | **English + Spanish** (`en`, `es`) from v1                                                                                              | User requirement; Astro built-in i18n routing                          |
| Props tables      | **Build script (`tooling/props-doc`)** using **ts-morph** on each framework package → `props/*.json` consumed by `<PropsTable>` in docs | One pipeline for all frameworks; prop names stay English (API surface) |
| LLM exports       | **`llms.txt` + per-component `.md`** generated at docs build — **one set per locale**                                                   | Already in brief; static output                                        |
| Theme Builder     | **In scope v1** — Svelte island at `/[lang]/theme-builder`                                                                              | See Theme Builder section below                                        |

### Docs i18n (en + es)

| Decision           | Choice                                                                                                              |
| ------------------ | ------------------------------------------------------------------------------------------------------------------- | --------------------- |
| Locales            | **`en`** (default), **`es`**                                                                                        |
| URL shape          | **`moderno.style/en/...`** and **`moderno.style/es/...`** — locale always in path                                   |
| Astro config       | Built-in **`i18n`** routing (`locales`, `defaultLocale: 'en'`, `prefixDefaultLocale: true`)                         |
| Content            | Content Collections with locale segment: `src/content/docs/en/`, `src/content/docs/es/` — same slug in both locales |
| UI chrome          | Shared layout strings in `apps/docs/src/i18n/en.json` and `es.json`; consumed by Astro layouts                      |
| Untranslated pages | Build fails if a slug exists in one locale but not the other (CI guard)                                             |
| Language switcher  | Header control linking the same slug across locales; `hreflang` + canonical per page                                |
| Code samples       | Installation commands and API identifiers stay **English** in both locales; only prose translates                   |
| Pagefind           | Build runs Pagefind twice (filter by `[data-pagefind-language="en                                                   | es"]`on`<html lang>`) |
| Props / types      | `<PropsTable>` labels translated via i18n JSON; prop names and types remain English                                 |

Component API docs do not get per-locale prop renames — only narrative content and UI chrome translate.

Docs deploy is tied to registry deploy: the same Vercel project serves MDX docs and the `/r/` registry static files from `apps/docs/public/r/` (copied or symlinked at build from `registry/`).

## Theme authoring and Theme Builder (v1)

Themes are **CSS-first registry items**, not npm packages and not edited inside components. **Theme Builder** is the primary authoring surface in v1 — not deferred.

### Source of truth pipeline

```
Theme Builder UI  →  tokens.dtcg.json  →  theme-compile  →  theme.css  →  registry item + CLI copy
       ↑______________________________|  (import existing registry theme as starting point)
```

| Decision               | Choice                                                                                                                                                                    |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Theme Builder location | **`apps/docs`** — route `/[lang]/theme-builder`, **Svelte island** (client-loaded)                                                                                        |
| Editable surface       | **Semantic contract slots** from CONTRACT.md (`--primary`, `--background`, `--radius`, …) + **brand primitives** (`--mod-*` ramps). Components are never edited.          |
| Live preview           | Preview panel applies CSS vars to a scoped container (`[data-theme-preview]`) rendering real primitives (Button, Input, Card, …) via existing docs Preview infrastructure |
| Light / dark           | Toggle edits **both** `:root` and `.dark` scopes; export includes both                                                                                                    |
| Multi-brand            | Optional `[data-brand="custom"]` block in export when user sets a brand id                                                                                                |
| Import                 | **Load base theme** from registry (`theme-moderno`, `theme-contrast`) or from pasted `tokens.dtcg.json`                                                                   |
| Session persistence    | **URL search params** (shareable link) + **localStorage** fallback — no server-side accounts in v1                                                                        |
| Export — CSS           | Download / copy **`theme.css`** (semantic aliases + OKLCH ramps)                                                                                                          |
| Export — DTCG          | Download / copy **`tokens.dtcg.json`**                                                                                                                                    |
| Export — CLI           | Copy install snippet: `moderno add theme-custom --from ./theme.css` (or path after download)                                                                              |
| Registry integration   | **Save to registry** = maintainer workflow: export → commit under `registry/themes/{name}/` → `pnpm theme:build` validates → ships in `/r/themes/{name}.json`             |
| Compile script         | **`pnpm theme:build`** (`tooling/theme-compile`) — shared by Theme Builder export and CI; validates contrast (WCAG AA warnings)                                           |
| Consumer install       | `moderno add theme-moderno` (or custom fork) copies `theme.css` + font assets into the consumer project                                                                   |
| Consumer customize     | Re-open in Theme Builder, edit exported JSON/CSS, or fork the registry item — never edit component source                                                                 |

### Theme Builder UX (v1 scope)

- Controls grouped by token category: color, typography, radius, spacing, motion, charts (`--chart-1…5`).
- OKLCH color pickers for ramps and semantic slots.
- Font stack selector (self-hosted `@font-face` blocks included in CSS export when fonts differ from default).
- **Contrast checker** inline for text/background pairs.
- **Reset to base theme** and **undo** (in-memory stack).

**Out of scope v1:** Figma Variables export, server-side saved themes / accounts, collaborative editing.

## Considered options

- **Turborepo:** rejected for v1 — pnpm workspaces sufficient for one maintainer.
- **English-only docs:** rejected — bilingual en/es required from launch.
- **Theme Builder deferred:** rejected — builder is the authoring path for custom themes in v1; hand-editing JSON/CSS remains supported for maintainers and power users.
- **Separate Theme Builder app:** rejected — lives in docs to reuse Preview, themes, and i18n; one deploy.
- **Private npm / GitHub Packages:** rejected for v1 — public DS benefits from npm install DX; registry copy model handles customization.
- **Algolia doc search:** rejected for v1 — Pagefind is free and static.
- **Starlight / Docusaurus:** rejected — brief requires custom minimal docs like shadcn.

## Consequences

- CI must publish npm packages and deploy docs (including `/r/`) on every release; registry and npm versions should be released together from the same Changesets PR when possible.
- Docs PRs that add a page in one locale must include the paired translation (or CI fails).
- Theme Builder and `tooling/theme-compile` share one schema — schema changes require updating builder controls, compile script, and the CONTRACT.md slot contract together.
- Props tables depend on the ts-morph script staying in sync with package exports; breaking export paths requires regenerating props JSON in CI.
- Fase 6 (docs) scope grows: i18n plumbing, Theme Builder island, and per-locale Pagefind indexes ship with the docs site.
