# PRD — Phase 6: Docs

> Component library in flat Astro, bilingual en/es, with a Theme Builder and the static registry served from the same deploy.

## Objective

Publish `apps/docs`: an Astro + MDX + Content Collections site, minimalist (shadcn reference, no Starlight), with live previews as Svelte islands, copy/copy-as-markdown, autogenerated PropsTable, install by package manager, a **Theme Builder** and the registry served under `/r/`. Bilingual (en/es) from v1.

## Scope

- Flat Astro + MDX + Content Collections. Islands in **Svelte** (the only client runtime).
- Docs-specific components:
  - `<Preview>` — Svelte island that renders the real interactive component.
  - `<Code>` — block with a **copy** button; **Preview / Code** tabs.
  - `<PropsTable>` — autogenerated at build by `tooling/props-doc` (ts-morph) → JSON per component.
  - Copy-as-markdown per page + `llms.txt` + `.md` per component, **one set per locale**.
  - Copy component — install command (`bunx`/`pnpm dlx`/`npx`) and/or source.
  - Install block with **bun / pnpm / npm** tabs.
- i18n en+es: `/en/...` and `/es/...` routes, parallel Content Collections, language switcher, build fails if the translated pair is missing. API and code identifiers in English in both.
- Search: **Pagefind** with a per-locale index (`pagefind-en`, `pagefind-es`).
- **Theme Builder** — `/[lang]/theme-builder`, Svelte island: edits semantic slots + brand primitives with live preview over real primitives; exports `theme.css`, `tokens.dtcg.json` and a CLI snippet. Persistence: URL params + localStorage. Shares the schema with `tooling/theme-compile`.
- Static registry: `/r/registry.json` and `/r/{type}/{name}.json` served from `apps/docs/public/r/` (copied/symlinked from `registry/` at build).
- Hosting: Vercel, domain `moderno.style`.

## Functional requirements

| #    | Requirement                                                                                                                                                    |
| ---- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| F6.1 | Each component page: Preview/Code tabs, interactive preview (Svelte island), working copy button.                                                              |
| F6.2 | `<PropsTable>` is generated at build from each framework package's types; prop names in English, column labels translated.                                     |
| F6.3 | Copy-as-markdown per page + `llms.txt` + `.md` per component generated per locale at build.                                                                    |
| F6.4 | Install block with bun/pnpm/npm tabs; correct commands per package manager.                                                                                    |
| F6.5 | i18n parity: every slug exists in en and es; CI fails if a translation is missing. The language switcher links the same slug; `hreflang` + canonical per page. |
| F6.6 | Pagefind indexes per locale; search does not mix languages.                                                                                                    |
| F6.7 | The Theme Builder edits both scopes (`:root` + `.dark`), live preview over real components, and exports valid `theme.css` + `tokens.dtcg.json` + CLI snippet.  |
| F6.8 | The Theme Builder imports base themes from the registry (`theme-moderno`, `theme-contrast`) or a pasted `tokens.dtcg.json`; inline WCAG AA contrast checker.   |
| F6.9 | The same deploy serves the MDX docs and `/r/` (static registry); the CLI can point to the per-PR preview registry.                                             |

## Deliverables

- `apps/docs/` deployable on Vercel (`moderno.style`).
- `tooling/props-doc/` (ts-morph → `props/*.json`).
- Docs components: Preview, Code, Tabs, PropsTable, install/copy blocks.
- Theme Builder (Svelte island) reusing `tooling/theme-compile`.
- i18n JSON (`en.json`/`es.json`) + parallel Content Collections.
- Build pipeline that copies `registry/` → `public/r/` and runs Pagefind per locale.

## Acceptance criteria (DoD)

- [ ] Minimalist docs with preview/code, copy and copy-as-markdown working.
- [ ] Available in English and Spanish with slug parity (CI guard active).
- [ ] PropsTable reflects the real types of the packages.
- [ ] The Theme Builder exports valid CSS + DTCG and the preview is consistent with real components.
- [ ] Pagefind search works per locale.
- [ ] `/r/registry.json` accessible from the deploy and consumable by `@moderno/cli`.
- [ ] Production deploy on `moderno.style` (or fallback Vercel URL until the domain is registered).

## Dependencies

- Phase 3 (Svelte components for previews + PropsTable).
- Phase 4 (charts to document).
- Phase 5 (registry items + `theme-compile` the Theme Builder reuses).

## Out of scope (v1)

- Figma Variables export, server-side saved themes / accounts, collaborative editing (per ADR-0001).
- Algolia (Pagefind is the v1 solution).
- Docs versioning / multi-version.
